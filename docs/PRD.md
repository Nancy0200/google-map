# PRD — 即時路況留言板（Road Bulletin）

> 版本：v1.1　｜　更新日期：2026-05-14　｜　語言：繁體中文

---

## 1. 專案概述

### 背景與動機

各大導航應用程式（Google Maps、Waze、Apple Maps 等）提供的路況資訊往往來源不一、更新延遲，導致駕駛人無法即時掌握前方真實的道路狀況。本專案希望透過「即時路況留言板」讓同路段的駕駛人或副駕駛能夠互相分享當前資訊，補足現有導航 App 的不足。

### 目標用戶

| 角色 | 描述 |
|------|------|
| **主駕駛（Driver）** | 正在駕車中，透過快速按鈕回報路況，以彈幕接收資訊 |
| **副駕駛（Passenger）** | 坐在乘客座，在留言板手動輸入留言並瀏覽路況 |
| **一般使用者** | 尚未上路，查詢即時留言作為出發參考 |

### 核心價值主張

- **安全第一**：圓形快速按鈕設計，讓主駕駛不需打字即可發送常用路況資訊
- **即時性**：留言透過 WebSocket 即時廣播，彈幕同步顯示在導航畫面上
- **社群共享**：所有用戶共同維護路況資訊，彌補官方資料的落差

## 2. 功能需求

### 2.1 單頁整合介面

**說明**：整個應用程式集中在單一頁面，分為左右兩個區塊：

```
┌─────────────────────────────────────────────────────────┐
│                                                         │
│   導航畫面（左側 75%）         │  留言板（右側 25%）    │
│                                │                        │
│   深色模擬地圖背景             │  社群留言板            │
│   彈幕訊息覆蓋於地圖上         │  ─────────────────     │
│                                │  👤 用戶A              │
│   ~~前方塞車，請注意！~~       │  前方塞車...           │
│   ~~前方有車禍~~               │  ─────────────────     │
│                                │  👤 用戶B              │
│                                │  施工中...             │
│   ● 主選單   📌 📌 📌         │  [輸入留言...]         │
└─────────────────────────────────────────────────────────┘
```

**使用者故事**：
- 作為 **主駕駛**，我希望 **在同一個畫面同時看到導航與路況留言彈幕**，以便 **不需切換頁面就能掌握路況**。
- 作為 **副駕駛**，我希望 **在右側留言板自由輸入與瀏覽留言**，以便 **即時回報與接收路況資訊**。

---

### 2.2 即時路況留言板（右側）

**說明**：右側 25% 為社群式留言板，依時間倒序排列，即時顯示所有用戶的路況回報。提供手動輸入框讓副駕駛或一般使用者自由發送留言。

**使用者故事**：
- 作為 **副駕駛**，我希望 **能手動輸入自訂路況訊息並發送**，以便 **回報快速按鈕未涵蓋的特殊狀況**。
- 作為 **一般使用者**，我希望 **查看即時留言串**，以便 **出發前判斷路況**。

---

### 2.3 彈幕顯示（導航畫面上）

**說明**：留言板中的訊息同步以彈幕形式橫向滾動顯示在左側導航畫面上，讓主駕駛無需看留言板也能掌握路況。

**使用者故事**：
- 作為 **主駕駛**，我希望 **在導航畫面上看到彈幕滾動的路況資訊**，以便 **在不移開視線的情況下接收路況**。

---

### 2.4 快速回報按鈕（左下角）

**說明**：導航畫面左下角設置圓形快速按鈕系統，分為**主選單**與**釘選按鈕**兩部分。

#### 主選單按鈕

圓形按鈕，點擊後向上展開選項面板，包含以下圖例：

| 圖例 | 說明 | 發送至留言板的內容 |
|------|------|--------------------|
| 🔴 紅燈 | 當前車速低於 30 km/h | 「目前車速緩慢（< 30 km/h），請注意壅塞。」 |
| 🟡 黃燈 | 當前車速 30–60 km/h | 「目前車速中等（30–60 km/h），行駛順暢。」 |
| 🟢 綠燈 | 當前車速高於 60 km/h | 「目前車速良好（> 60 km/h），道路暢通。」 |
| 🚗💥 前方車禍 | 前方發生事故 | 「前方有車禍，請減速並注意安全！」 |
| 📦⚠️ 前方掉落物 | 路面有障礙物 | 「前方路面有掉落物，請小心閃避！」 |

> **速度選擇邏輯**：三個燈號圖例由駕駛依據當前時速手動點選，點擊後立即發送對應訊息至留言板。

#### 釘選按鈕（主選單右側）

- 主選單右側固定顯示 **3 個釘選格**
- 使用者可從主選單選項中，長按或透過設定將常用圖示釘選至此
- 釘選設定儲存於 `localStorage`，重新整理後保留
- 點擊釘選按鈕直接發送，無需展開主選單

**使用者故事**：
- 作為 **主駕駛**，我希望 **透過單一點擊快速回報路況**，以便 **行車時不需分心輸入文字**。
- 作為 **主駕駛**，我希望 **將最常用的按鈕釘選在主選單旁**，以便 **不需每次展開選單**。

---

### 2.5 留言過濾與分類

**說明**：留言板支援依類型篩選，讓使用者快速找到所需資訊。

**使用者故事**：
- 作為 **一般使用者**，我希望 **篩選特定類型的路況留言**，以便 **只查看與我相關的資訊**。

## 3. 非功能需求

### 3.1 技術限制

| 項目 | 規格 |
|------|------|
| 後端框架 | Python Flask |
| 模板引擎 | Jinja2 |
| 資料庫 | SQLite |
| 即時通訊 | Flask-SocketIO（WebSocket） |
| 前端 | HTML + CSS + Vanilla JavaScript |
| 地圖呈現 | 深色模擬地圖背景（CSS 實作，不整合外部地圖 API） |

### 3.2 效能考量

- 留言使用 WebSocket 即時推送，不依賴輪詢
- 資料庫僅保留最近 **24 小時**的留言，避免資料量過大
- 頁面首次載入時間應在 **2 秒**以內

### 3.3 安全考量

- 輸入內容需進行 **XSS 過濾**，防止惡意腳本注入
- 快速回報按鈕需有 **冷卻時間（Cooldown）**，防止洗版（同一裝置 60 秒內限發 3 則）
- 留言長度限制：**最多 100 字**

## 4. MVP 範圍

### Must Have（一定要有）

- [x] 單頁整合介面（左側導航 + 右側留言板）
- [x] 即時路況留言板（社群式，WebSocket 即時更新）
- [x] 彈幕顯示（留言同步覆蓋於導航畫面）
- [x] 快速回報主選單（速度燈號 + 車禍 + 掉落物圖例）
- [x] 釘選按鈕（3 格，儲存至 localStorage）
- [x] 留言儲存至 SQLite 資料庫

### Should Have（應該要有）

- [ ] 留言類型標籤（速度 / 車禍 / 掉落物 / 其他）
- [ ] 發送冷卻時間機制
- [ ] 彈幕動畫效果（CSS 水平滾動）

### Nice to Have（有更好）

- [ ] 真實地圖整合（Google Maps 或 OpenStreetMap）
- [ ] 留言按讚 / 確認功能（其他人驗證路況真實性）
- [ ] 使用者暱稱設定
- [ ] 推播通知（前方有新留言時提示）

|------|------|----------|
| 主頁面（唯一頁面） | `/` | 左側導航畫面 + 右側留言板 + 快速按鈕 |
