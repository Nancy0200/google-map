# PRD — 即時路況留言板（Road Bulletin）

> 版本：v1.0　｜　撰寫日期：2026-05-07　｜　語言：繁體中文

---

## 1. 專案概述

### 背景與動機

各大導航應用程式（Google Maps、Waze、Apple Maps 等）提供的路況資訊往往來源不一、更新延遲，導致駕駛人無法即時掌握前方真實的道路狀況。本專案希望透過「即時路況留言板」讓同路段的駕駛人或副駕駛能夠互相分享當前資訊，補足現有導航 App 的不足。

### 目標用戶

| 角色 | 描述 |
|------|------|
| **主駕駛（Driver）** | 正在駕車中，需要快速、安全地回報或接收路況資訊 |
| **副駕駛（Passenger）** | 坐在乘客座，可自由瀏覽與手動輸入路況留言 |
| **一般使用者** | 尚未上路，查詢特定路段的即時留言作為出發參考 |

### 核心價值主張

- **安全第一**：提供一鍵快速回報按鈕，讓主駕駛不需打字即可發送常用路況資訊
- **即時性**：留言即時顯示於留言板，所有人立刻看到最新動態
- **社群共享**：讓同路段的所有用戶共同維護路況資訊，彌補官方資料的落差

## 2. 功能需求

### 2.1 即時路況留言板

**說明**：顯示當前路段（或全體）的即時路況留言串，依時間倒序排列，並以彈幕方式呈現滾動效果。

**使用者故事**：
- 作為 **副駕駛**，我希望 **能夠即時看到其他用戶回報的路況留言**，以便 **提前提醒駕駛人注意前方狀況**。
- 作為 **一般使用者**，我希望 **在出發前查看特定路段的最新留言**，以便 **選擇最佳路線**。

---

### 2.2 快速回報按鈕（一鍵發送）

**說明**：提供數個預設路況字串按鈕，主駕駛只需點擊即可立即發送，不需手動輸入文字，降低行車危險。

**預設快速回報選項（可擴充）**：

| 按鈕標籤 | 發送內容 |
|----------|----------|
| 🚗 前方塞車 | 「前方塞車，請注意！」 |
| ⚠️ 突發狀況 | 「此路段有突發狀況，請小心行駛！」 |
| 🐢 車速過慢 | 「此路段時速過低，建議改道。」 |
| 🚧 道路施工 | 「前方道路施工，請減速慢行。」 |
| 🚔 警察執法 | 「前方有警察執法，請注意速限。」 |

**使用者故事**：
- 作為 **主駕駛**，我希望 **透過單一點擊快速回報路況**，以便 **在不分心的情況下提供路況資訊給其他人**。

---

### 2.3 副駕駛互動頁面

**說明**：專為副駕駛設計的獨立頁面，提供完整的手動輸入留言功能，並顯示彈幕式路況資訊流。

**使用者故事**：
- 作為 **副駕駛**，我希望 **能夠手動輸入自訂路況訊息並發送**，以便 **回報按鈕未涵蓋的特殊狀況**。
- 作為 **副駕駛**，我希望 **在螢幕上看到彈幕滾動顯示的路況留言**，以便 **快速掌握路況全貌**。

---

### 2.4 留言歷史記錄

**說明**：保存近期路況留言供查詢，包含留言時間、發送者角色（駕駛 / 乘客）。

**使用者故事**：
- 作為 **一般使用者**，我希望 **查看過去一小時內的路況留言記錄**，以便 **判斷路段是否仍有狀況**。

---

### 2.5 留言過濾與分類

**說明**：依類型（塞車、施工、突發、測速）過濾留言，讓使用者快速找到所需資訊。

**使用者故事**：
- 作為 **一般使用者**，我希望 **篩選特定類型的路況留言**，以便 **只查看與我相關的資訊**。

## 3. 非功能需求

### 3.1 技術限制

| 項目 | 規格 |
|------|------|
| 後端框架 | Python Flask |
| 模板引擎 | Jinja2 |
| 資料庫 | SQLite |
| 即時通訊 | Flask-SocketIO（WebSocket） |
| 前端 | HTML + CSS + Vanilla JavaScript |

### 3.2 效能考量

- 留言板每 **5 秒**自動刷新（或使用 WebSocket 即時推送）
- 資料庫僅保留最近 **24 小時**的留言，避免資料量過大
- 頁面首次載入時間應在 **2 秒**以內

### 3.3 安全考量

- 輸入內容需進行 **XSS 過濾**，防止惡意腳本注入
- 快速回報按鈕需有 **冷卻時間（Cooldown）**，防止洗版（同一裝置 60 秒內限發 3 則）
- 留言長度限制：**最多 100 字**
